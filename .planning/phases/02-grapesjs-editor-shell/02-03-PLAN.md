    Create `components/editor/TopBar.tsx` as a 'use client' component (it uses useState and calls editor methods imperatively).

    Props interface:
    ```typescript
    interface TopBarProps {
      editorRef: React.RefObject<Editor | null>
      userEmail: string
    }
    ```

    Implementation requirements:
    1. Import `Editor` type from 'grapesjs' (type-only import)
    2. Import `LogoutButton` from '@/components/auth/LogoutButton'
    3. `useState<'desktop' | 'mobile'>('desktop')` — tracks active device for button highlight
    4. Undo button: `onClick={() => editorRef.current?.runCommand('core:undo')}` — use optional chaining, editor may not be set yet
    5. Redo button: `onClick={() => editorRef.current?.runCommand('core:redo')}` — same
    6. Desktop button: `onClick={() => { editorRef.current?.setDevice('Desktop'); setActiveDevice('desktop') }}`
    7. Mobile button: `onClick={() => { editorRef.current?.setDevice('Mobile'); setActiveDevice('mobile') }}`
    8. Active device button should have a highlighted style (darker background or ring) vs inactive (neutral)
    9. Render user email as a small text label (`<span>` or `<p>` with text-sm text-gray-600)
    10. Render `<LogoutButton />` next to the email

    Layout (left to right in the top bar):
    - LEFT: App name or logo (simple text "AI Content Booster" or equivalent)
    - CENTER: Undo | Redo | Desktop | Mobile buttons
    - RIGHT: User email + LogoutButton

    Use Tailwind CSS 4 utilities. Suggested classes:
    - Bar container: `flex items-center justify-between px-4 py-2 bg-white border-b border-gray-200 h-12`
    - Buttons: `px-3 py-1 text-sm rounded` with active state: `bg-blue-100 text-blue-700` and inactive: `text-gray-600 hover:bg-gray-100`
    - Separator between button groups: a thin `|` character or `<div className="w-px h-5 bg-gray-300 mx-2" />`

    NOTE on keyboard shortcuts (ED-03): GrapesJS built-in UndoManager automatically handles Ctrl+Z / Ctrl+Y keyboard shortcuts once GrapesJS is mounted. The toolbar buttons are ADDITIONAL controls, not replacements. Do NOT add manual keydown event listeners in TopBar — GrapesJS handles keyboard shortcuts natively.

    The component uses `React.RefObject` (not `MutableRefObject`) for the editorRef prop — the ref is created in the parent with `useRef` and passed down. Optional chaining (`?.`) handles the case where the editor hasn't initialized yet (before onEditor fires).

    Export as default export.
