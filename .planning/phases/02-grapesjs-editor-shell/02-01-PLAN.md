  <action>
    Create `lib/mockBlock.ts` exporting a GrapesJS project data constant named `MOCK_BLOCK`.

    The structure must be a valid argument to `editor.loadProjectData()`. GrapesJS 0.22 project data format:
    ```
    {
      assets: [],
      styles: [],
      pages: [
        {
          frames: [
            {
              component: {
                type: 'wrapper',
                components: [ ...child components... ]
              }
            }
          ]
        }
      ]
    }
    ```

    The root wrapper must contain one `section` component (draggable container) wrapping three individually-draggable children:
    1. An `h2` text component — content: `"Tiêu đề quảng cáo chính"`
    2. A `p` text component — content: `"Mô tả sản phẩm với nội dung tiếng Việt của bạn tại đây."`
    3. A `button` text component — content: `"Mua ngay"`

    Important structural requirements for ED-02 (drag reorder) to be testable:
    - The h2, p, and button must be SIBLINGS within the section (not nested) so they can be individually selected and dragged to reorder
    - Each child component must have `draggable: true` and editable text (type: 'text')

    Style guidance (use kebab-case CSS property names in GrapesJS style objects):
    - section: padding 40px 24px, background #f8f9fa, text-align center, font-family Arial sans-serif
    - h2: font-size 28px, font-weight bold, color #1a1a1a, margin-bottom 12px, margin-top 0
    - p: font-size 16px, color #444444, margin-bottom 24px, line-height 1.6
    - button: background-color #e53e3e, color #ffffff, padding 12px 32px, border none, border-radius 4px, font-size 16px, font-weight bold, cursor pointer

    No images in this mock (per CLAUDE.md "No base64 images" and D-12 simple content requirement).

    Type the export: `export const MOCK_BLOCK = { ... } as const` — the `as const` allows TypeScript to infer the shape without needing an explicit type import.

    After creating the file, run `npm run build` to confirm TypeScript accepts the file.
