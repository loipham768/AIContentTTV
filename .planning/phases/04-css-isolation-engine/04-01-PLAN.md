  <name>Task 2: Create lib/cssIsolation.ts</name>
  <files>lib/cssIsolation.ts</files>
  <action>
    Create `lib/cssIsolation.ts` with the following exact content.

    ```typescript
    import juice from 'juice'

    // Browser-only: uses DOMParser — import only from 'use client' components
    export function isolateCss(html: string, css: string): string {
      const inlined = juice.inlineContent(html, css)

      const parser = new DOMParser()
      const doc = parser.parseFromString(inlined, 'text/html')

      doc.querySelectorAll('script').forEach(el => el.remove())

      doc.body.querySelectorAll('*').forEach(el => {
        el.removeAttribute('class')
        Array.from(el.attributes)
          .filter(attr => attr.name.startsWith('data-'))
          .forEach(attr => el.removeAttribute(attr.name))
      })

      return doc.body.innerHTML
    }
    ```

    Implementation notes for the executor:
    - `juice.inlineContent(html, css)` merges CSS rules from `css` into `style=""` attributes on matching elements in `html`. If `css` is empty (which it is for our inline-style blocks), it returns `html` unchanged.
    - `DOMParser.parseFromString(html, 'text/html')` creates a full document; `doc.body` holds the parsed content; `doc.body.innerHTML` returns it as a string.
    - `Array.from(el.attributes)` creates a snapshot array before iteration — required because `el.attributes` is a live NamedNodeMap and removing attributes while iterating it would skip entries.
    - `data-*` attributes include GrapesJS internals like `data-gjs-type`, `data-gjs-highlightable`, etc. Removing all `data-*` attributes is the correct broad-sweep approach.
    - Do NOT add a `'use client'` directive — this is a library file, not a React component.
    - Do NOT add try/catch — errors propagate to the caller (TopBar) where they are caught.

    After creating the file, run `npx tsc --noEmit` and fix any TypeScript errors before marking this task done.
