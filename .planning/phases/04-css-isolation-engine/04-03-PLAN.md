    Run `npm run build` to verify the entire project compiles without errors. This is the critical bundling gate — webpack (used in production) resolves `juice`'s browser-compatible build differently from Turbopack (used in dev). A clean build confirms juice bundles correctly for the browser.

    Expected: Build exits 0. All pages compile. No red errors in the output.

    Common failure modes and how to fix them:
    - **juice bundling error** (e.g., "cannot resolve 'fs'" or "cannot resolve 'path'"): juice is importing a Node.js module that Next.js can't bundle for the browser. Fix: create `next.config.ts` (or update existing) with `webpack: (config) => { config.resolve.fallback = { fs: false, path: false }; return config }`. If that doesn't work, replace `import juice from 'juice'` with the DOM-only fallback below.
    - **TypeScript error in lib/cssIsolation.ts**: fix the specific error in that file, re-run `npm run build`.
    - **TypeScript error in components/editor/TopBar.tsx**: fix the specific error, re-run `npm run build`.
    - **Any other missing module**: read the error and install the missing package.

    DOM-only fallback for juice bundling failure (use only if juice fails to bundle — this skips class-based CSS inlining, which is acceptable because our blocks use inline styles):
    ```typescript
    // Replace the entire lib/cssIsolation.ts with this if juice cannot bundle:
    // Browser-only: uses DOMParser — import only from 'use client' components
    export function isolateCss(html: string, css: string): string {
      const parser = new DOMParser()
      const doc = parser.parseFromString(html, 'text/html')

      doc.querySelectorAll('script').forEach(el => el.remove())

      doc.body.querySelectorAll('*').forEach(el => {
        el.removeAttribute('class')
        Array.from(el.attributes)
          .filter(attr => attr.name.startsWith('data-'))
          .forEach(attr => el.removeAttribute(attr.name))
      })

      return doc.body.innerHTML
    }
    ```
    If using the fallback, uninstall juice: `npm uninstall juice && npm uninstall -D @types/juice`

    Do NOT use `npm run build -- --no-lint` to mask errors. Fix the actual issue.
