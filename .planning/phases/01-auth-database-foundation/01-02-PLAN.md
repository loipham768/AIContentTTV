<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser form → POST /api/auth/register | User-controlled email + password crosses trust boundary into registration API |
| Registration API → MongoDB Atlas | User data written to DB; must be validated and hashed before write |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | `app/api/auth/register/route.ts` | mitigate | Duplicate email error is single generic message "Email này đã được sử dụng" — does NOT confirm whether the email is registered, only that it cannot be used again (slight disclosure acceptable for UX; enumeration attack requires knowing email in advance) |
| T-02-02 | Information Disclosure | `app/api/auth/register/route.ts` | mitigate | `bcrypt.hash(password, 12)` — password never stored in plaintext; $2b$ prefix verified in tests; per D-21 and ASVS V6 |
| T-02-03 | Tampering | `app/api/auth/register/route.ts` | mitigate | Zod `registerSchema.safeParse()` validates all input before DB access; rejects non-string values including MongoDB operator injection |
| T-02-04 | Elevation of Privilege | `components/auth/LoginRegisterCard.tsx` | mitigate | Client-side Zod validation (D-13) is defense-in-depth only — server always re-validates; client validation cannot be bypassed to skip server-side checks |
| T-02-05 | Spoofing | `components/auth/LoginRegisterCard.tsx` handleLogin | mitigate | `signIn('credentials', { redirect: false })` — error is mapped to generic Vietnamese string on client; raw 'CredentialsSignin' error code from NextAuth never shown to user (Pitfall 2 fix) |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this plan; all packages from Plan 01 already audited |
</threat_model>
