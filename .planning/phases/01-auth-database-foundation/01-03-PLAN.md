<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → NextAuth signIn | Credentials flow crosses from client to NextAuth authorize() — must never leak email enumeration |
| NextAuth JWT → Browser cookie | Session token stored in HttpOnly cookie — client cannot read or tamper with JWT |
| Protected /editor → auth() | Server Component must independently verify session (belt-and-suspenders) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | `LoginRegisterCard.tsx` login handler | mitigate | `result.error` is NEVER displayed raw; always mapped to generic Vietnamese string "Email hoặc mật khẩu không đúng" (D-10, Pitfall 2 fix) — no email enumeration |
| T-03-02 | Elevation of Privilege | `LoginRegisterCard.tsx` callbackUrl | mitigate | callbackUrl comes from server-passed prop (from searchParams), not from `window.location` — prevents open redirect to external domains (NextAuth validates callbackUrl origin by default) |
| T-03-03 | Elevation of Privilege | `app/editor/page.tsx` | mitigate | Server Component independently calls `auth()` even though middleware.ts also checks — belt-and-suspenders (D-06); LogoutButton added but does not bypass auth check |
| T-03-04 | Spoofing | `LogoutButton.tsx` signOut | mitigate | `signOut({ callbackUrl: '/login' })` clears HttpOnly JWT cookie server-side; client cannot forge a signout that leaves cookie intact |
| T-03-05 | Elevation of Privilege | Session JWT | mitigate | NextAuth JWT strategy signs tokens with AUTH_SECRET (≥ 32 chars, generated via `npx auth secret`); tampered tokens fail signature verification and return null session |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this plan; all from Plan 01 Package Legitimacy Audit |
</threat_model>
