<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → API | Unauthenticated HTTP requests cross from untrusted client to Next.js route handlers |
| Next.js edge → Node.js | middleware.ts runs before route handlers; must not contain Node.js-only code |
| Next.js → MongoDB Atlas | dbConnect() crosses network boundary to database; credentials in env vars |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | `auth.ts` Credentials authorize() | mitigate | Returns null (not custom error) for any auth failure; NextAuth maps to generic CredentialsSignin error — prevents caller from learning whether email exists (D-10) |
| T-01-02 | Information Disclosure | `auth.ts` authorize() | mitigate | Never log password parameter in authorize(); return null silently — no error message forwarded to client |
| T-01-03 | Information Disclosure | MongoDB Atlas | mitigate | MONGODB_URI in .env.local (gitignored); never committed to version control |
| T-01-04 | Tampering | `lib/mongodb.ts` | mitigate | Mongoose schema types reject BSON operators (e.g., `$gt`) in string fields; Zod pre-validation strips non-string inputs before any DB call |
| T-01-05 | Elevation of Privilege | `middleware.ts` | mitigate | auth as middleware intercepts all /editor and /api/* requests; unauthenticated requests never reach route handlers |
| T-01-06 | Elevation of Privilege | `app/editor/page.tsx` | mitigate | Belt-and-suspenders: Server Component calls auth() independently even if middleware were misconfigured (D-06) |
| T-01-07 | Denial of Service | `lib/mongodb.ts` | mitigate | maxPoolSize: 10 caps Atlas connections; singleton prevents new connection per invocation |
| T-01-08 | Tampering | npm package installs | mitigate | Package Legitimacy Audit in RESEARCH.md confirms all packages approved — no [ASSUMED] or [SUS] packages; slopcheck false-positives documented |
| T-01-SC | Tampering | npm/pip/cargo installs | mitigate | All packages verified via npm registry + source repo confirmation in RESEARCH.md Package Legitimacy Audit; no postinstall scripts; all disposition: Approved |
</threat_model>
